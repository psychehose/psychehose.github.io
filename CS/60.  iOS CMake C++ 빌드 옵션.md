

*  -fobjc-abi-version=2
	*   의미: Objective-C의 Modern ABI (Application Binary Interface) 버전 2를 사용함
    *   최신 iOS 및 64-bit macOS 환경의 표준 규약
    *   C++ SDK가 Apple의 시스템 프레임워크(Foundation, UIKit 등)와 데이터 구조 및 함수 호출 규약을 일치시켜 충돌 없이 통신하기 위해 필수적

*  -fembed-bitcode=off
	*  의미: Apple의 Bitcode(중간 코드) 생성 기능을 비활성화
	*  Xcode 14부터 Apple이 Bitcode 지원을 중단

* -fno-rtti
	*  의미: C++ 표준 기능인 RTTI (Run-Time Type Information) 끔 
	* `dynamic_cast`, `typeid` 사용 불가
	* unreal engine은 RTTI 사용 안해서 끔

* -fvisibility=hidden
	*   의미: 라이브러리 내부의 심볼(함수/변수 이름)을 기본적으로 숨김
    *   심볼 충돌 방지: SDK 내부에서 사용하는 라이브러리(예: OpenSSL)와 언리얼 엔진이 사용하는 라이브러리의 이름이 겹쳐서 발생하는 크래시를 방지
    *   명시적으로 공개(`export`)한 API만 외부에서 접근 가능하게 하여 모듈성을 강화

* -fstack-protector-strong
*   의미: 스택 메모리에 카나리아(Canary) 값을 심어 버퍼 오버플로우 공격을 감지하고 차단
	*   앱 보안성을 높이기 위한 필수 옵션으로, 해킹 시도 시 앱을 강제 종료하여 시스템을 보호

* -fobjc-call-cxx-cdtors
	*  의미: Objective-C 객체의 생명주기에 맞춰 멤버로 포함된 C++ 객체의 생성자/소멸자를 자동으로 호출함
    *   Objective-C++ (.mm) 혼용 환경에서 C++ 객체가 올바르게 초기화되고 해제되도록 보장
    *   이 옵션이 없으면 소멸자가 호출되지 않아 메모리 누수가 발생할 수 있음

* -ffunction-sections & -fdata-sections
	*   의미: 각 함수와 데이터를 개별 섹션으로 분리하여 컴파일
    *   Dead Code Stripping: 링킹(Linking) 단계에서 실제로 사용되지 않는 함수나 데이터를 링커가 식별하여 최종 앱 바이너리에서 제거할 수있음
    *   앱 용량(IPA Size)이 줄어듦
